# Repo Reality Report

This report is the human-readable audit companion to the reproducible inventory
generated by `make repo-reality-report`. The generated inventories live under
`build/repo-reality/` and should be refreshed whenever new remote features,
permissions, or workflows are added.

## How to regenerate the evidence
- Run `make repo-reality-report` to refresh:
  - `build/repo-reality/network_endpoints.txt`
  - `build/repo-reality/permissions_inventory.txt`
  - `build/repo-reality/dependency_inventory.txt`
  - `build/repo-reality/workflow_lint.txt`
- Run `make workflow-lint` locally for a direct GitHub Actions syntax check.

## Non-negotiable repo policy
- No app-level account is required for the free core.
- Prayer times, adhan scheduling, qibla bearing, Hijri date, and Quran Arabic must keep working without a server.
- Optional infrastructure is allowed only for optional features such as remote Hadith packs or store billing.
- Analytics and telemetry remain disabled by default.

## Network surfaces
The repo currently contains these intentional remote surfaces:

- Remote Hadith packs:
  - `features/hadith/lib/src/data/hadith_pack_remote_data_source.dart`
  - `services/content-pack-worker/src/index.mjs`
  - Purpose: fetch manifest, access grants, and pack downloads for optional Sunni Hadith packs
- RevenueCat billing:
  - `mobile/app/lib/src/bootstrap/revenuecat_mobile_subscription_provider.dart`
  - `services/content-pack-worker/src/lib/revenuecat.mjs`
  - Purpose: optional entitlement sync and paid-pack authorization
- Quran translation catalog and verse downloads:
  - `features/quran/lib/src/data/quran_enc_remote_data_source.dart`
  - Purpose: optional translation catalog refresh and translation downloads
- Legacy / migration Hadith API adapter:
  - `features/hadith/lib/src/data/hadeethenc_remote_data_source.dart`
  - Purpose: migration/admin path, not the primary starter-pack flow
- Scholar feed refresh:
  - `features/scholar_feed/lib/src/data/scholar_feed_defaults.dart`
  - Purpose: optional RSS sync for public scholar sources
- AI assistant BYOK calls:
  - `features/ai_assistant/lib/src/data/openai_responses_client.dart`

## Identifier generation, storage, and transmission
- App user identity for optional RevenueCat/Hadith pack flows:
  - generated in `mobile/app/lib/src/bootstrap/app_identity_store.dart`
  - stored in secure OS storage after this hardening pass
  - created lazily when optional billing or remote Hadith pack flows need it
  - transmitted only to RevenueCat-backed billing or remote Hadith pack access flows
- BYOK OpenAI API key:
  - stored in secure OS storage via `mobile/app/lib/src/bootstrap/flutter_secure_value_store.dart`
- Profile/settings state:
  - stored locally via `mobile/app/lib/src/bootstrap/app_profile_store.dart`

## Notification scheduling flow
- App bootstrap initializes local notifications and syncs the prayer schedule:
  - `mobile/app/lib/src/bootstrap/app_controller.dart`
  - `mobile/app/lib/src/bootstrap/app_notification_sync_service.dart`
  - `mobile/app/lib/src/bootstrap/local_notifications_service.dart`
- iOS scheduling intentionally stays below the 64 pending-notification cap:
  - `features/prayer/lib/src/services/ios_notification_window_planner.dart`
  - current safety window: schedule up to 60 reminders with a 4-notification buffer
- Android exact/inexact alarm policy:
  - `features/prayer/lib/src/services/android_alarm_policy.dart`
- Android reboot rescheduling uses the local notifications boot receiver:
  - `mobile/app/android/app/src/main/AndroidManifest.xml`

## Content-pack trust boundaries
- Mobile app:
  - never needs the Hadith Worker for prayer/qibla/Quran Arabic
  - requests remote Hadith packs only when the user explicitly enters that feature and installs packs
- Cloudflare Worker:
  - serves manifest + signed download access
  - validates exact pack/object mapping before download
- R2:
  - stores remote Hadith pack archives only
- KV:
  - stores starter-pack claims and daily budget counters
- RevenueCat:
  - optional paid entitlement authority for extra language packs
