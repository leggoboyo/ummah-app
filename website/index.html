Mission

Build an Islamic app that is easier to trust because it asks less of the user

What Ummah App is trying to do

The app is being built around a small set of hard promises: the free worship core should stay usable offline, the content should stay attributable, optional cloud features should stay optional, and the product should avoid becoming a surveillance-heavy or ad-shaped experience.

That means the repo is structured to separate the local essentials from optional packs, optional billing, and optional AI. When something needs a network, that boundary is supposed to be visible in both the code and the docs.

Commitments that drive decisions

  • No required app-level account for the free core
  • No analytics or telemetry by default
  • No hidden server dependency for prayer, qibla, Hijri, or Quran Arabic
  • Keep religious sources versioned, attributable, and inspectable

Things the project is explicitly trying to avoid

  • Turning worship utilities into an engagement funnel
  • Bundling every large corpus into the first install
  • Shipping black-box religious output without visible sourcing
  • Letting optional infrastructure quietly become mandatory

Design principles

Low monetary cost, low performance cost, low hidden behavior

Stay useful offline

The free core is designed so it does not need a backend for prayer calculations, local reminders after scheduling, qibla bearing, Hijri date, or Quran Arabic reading.

Stay inspectable

Religious source pages, repo audits, privacy docs, security docs, and pack source/version docs are all linked below so people can verify the claims instead of trusting marketing language.

Stay light

Optional features are modular. Packs download on demand. The public site itself is plain static HTML, CSS, and tiny JavaScript so it can live on GitHub Pages with almost no operational cost.

Low-cost posture

Designed to avoid both surprise bills and sluggish app behavior

Low monetary cost

  • No required backend for the free worship core.
  • Optional Hadith infrastructure is scoped to packs only.
  • BYOK AI keeps inference optional and founder cost near zero.
  • This public site is static and GitHub Pages-friendly.

Low performance cost

  • Large study content is modular instead of bundled by default.
  • Startup should not block on optional remote services.
  • On-device search and local SQLite keep repeated use fast.
  • No analytics or ad SDKs are in the current app manifests.
